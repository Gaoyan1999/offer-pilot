# OfferPilot One-Day MVP Plan

## Summary

OfferPilot is a free-form job search AI agent. Users can upload a resume as PDF, Markdown, or plain text, or simply describe their background and target role. If the agent does not have enough information, it asks follow-up questions. Once the profile is complete enough, it searches for jobs in the user's target location and presents the results in a dedicated job list UI.

After the user selects a specific job, the agent tailors the user's resume based on verified facts and generates a role-specific PDF resume using a fixed template.

The MVP focuses on two core features:

1. Automatically finding relevant jobs.
2. Generating a tailored PDF resume for a selected job.

The MVP does not include automatic job applications.

## Product Shape

The main interface should use a hybrid format: a free-form chat/input experience plus structured results.

- The free-form input area lets users describe their needs naturally.
- The upload area supports PDF resumes, Markdown resumes, and plain text.
- The agent can ask follow-up questions when key information is missing, such as target location, role type, years of experience, remote preference, or work authorization constraints.
- Job results should not be shown as chat bubbles. They should use a dedicated job list UI.
- After a user selects a job, the detail area should show the job description, match reasoning, gaps, risks, and resume generation entry point.

This format keeps the product flexible while making the demo value easy to understand during a hackathon presentation.

## Core Feature 1: Automatic Job Discovery

### Input

Users can provide one or more of the following:

- PDF resume
- Markdown resume
- Plain text resume
- Short background description
- Target role
- Target location
- Remote, hybrid, or onsite preference
- Work authorization or visa constraints

### Agent Behavior

The agent first parses the user input and creates a candidate profile:

- Skills
- Work experience
- Project experience
- Years of experience
- Target role direction
- Target location
- Job preferences
- Possible constraints

If required information is missing, the agent should not start searching immediately. It should ask follow-up questions first.

Example follow-up questions:

- "Which city or country do you want to search in?"
- "Are you targeting frontend, backend, full-stack, AI engineer, or product engineer roles?"
- "Are you open to remote roles?"
- "How many years of experience do you have?"

### Job Search

Job search should use a "search API + fallback mock data" strategy.

Primary path:

- Use a search API to fetch real job results.
- Search queries are generated by the agent based on the candidate profile.
- Each result should include job title, company, location, job URL, and a short job description.

Fallback path:

- Prepare a small mock job dataset.
- Use mock jobs when the search API fails, times out, returns no results, or returns low-quality results.

### Job List UI

Jobs should be displayed in a list or table, not inside chat bubbles.

Each job item should show:

- Job title
- Company
- Location
- Source
- Job URL
- Match score
- Match rationale
- Missing skills or risks

By default, jobs should be sorted by match score and show the top matches first.

## Core Feature 2: Tailored PDF Resume

### Flow

1. The user selects a specific job from the job list.
2. The agent reads the selected job description.
3. The agent compares the candidate profile against the job description.
4. The agent rewrites and reorders resume content based on verified user facts.
5. The system generates a PDF resume using a fixed template.

### Resume Tailoring Rules

Resume tailoring must be fact-based.

Allowed:

- Rewrite the professional summary to better match the target role.
- Reorder skills based on relevance to the job description.
- Highlight relevant projects, experience, and keywords.
- Rewrite bullet points for clearer positioning.
- Remove or de-emphasize less relevant content.
- Provide resume improvement suggestions.

Not allowed:

- Fabricating companies, titles, education, projects, or dates.
- Fabricating skills the user did not provide.
- Fabricating quantified outcomes.
- Claiming the user has done something without evidence from the provided resume or description.

If the job requires important experience that the user has not provided, the agent should explicitly identify the gap instead of inventing it.

### PDF Output

The PDF should use a fixed resume template. The MVP should not attempt to preserve or reproduce the original PDF layout.

The output should include:

- Name and contact information
- Professional summary
- Skills
- Experience
- Projects
- Education
- Optional tailored application summary

For demo convenience, the system can also show a Markdown or HTML preview before PDF generation.

## Agent Flow

Recommended steps:

1. `parse_user_profile`
   - Read uploaded files and text input.
   - Extract the candidate profile.

2. `identify_missing_info`
   - Check whether key job-search information is missing.
   - If information is missing, generate follow-up questions.

3. `search_jobs`
   - Generate search queries from the candidate profile.
   - Call the search API.
   - Fall back to mock jobs when needed.

4. `rank_jobs`
   - Score each job against the candidate profile.
   - Generate match rationale, gaps, and risks.

5. `tailor_resume`
   - Generate tailored resume content for the selected job.
   - Ensure all resume content is based on verified user facts.

6. `render_resume_pdf`
   - Render the tailored resume using a fixed template.
   - Return a preview and downloadable PDF.

## Demo Script

1. The user uploads a resume or enters a short background description.
2. The user says: "I want AI product engineer roles in Sydney or remote."
3. The agent asks a follow-up question if important information is missing, such as years of experience or target location.
4. The user provides the missing information.
5. The agent starts searching for jobs.
6. The page shows a job list with match scores.
7. The user clicks one job.
8. The agent shows job details, match rationale, and gaps.
9. The user clicks generate tailored resume.
10. The system generates a template-based PDF resume.

## Test Plan

- Test PDF, Markdown, plain text, and short description inputs.
- Test whether the agent asks follow-up questions when information is missing.
- Test whether the agent starts job search when enough information is available.
- Test fallback to mock jobs when the search API fails.
- Test whether the job list is sorted by match score.
- Test whether selecting a job can generate a tailored resume.
- Test that the generated resume does not fabricate experience.
- Test that the PDF can be downloaded and opened successfully.

## One-Day Build Priority

### Must Have

- Free-form input and resume upload entry point.
- Follow-up questions when information is missing.
- Job search or mock job search.
- Job list UI.
- Match scoring and rationale.
- Tailored resume generation for a selected job.
- PDF export.

### Nice To Have

- Agent execution trace.
- Job filters.
- Cover letter.
- Recruiter outreach message.
- Apply queue.
- Resume version history.

## Assumptions

- The MVP only covers job discovery and tailored resume generation.
- The MVP does not perform real automatic applications.
- Job search uses a "search API + fallback mock data" strategy.
- PDF output uses a fixed template.
- The main interaction model is "free-form input + structured job list."
- Resume tailoring must be strictly based on user-provided facts.
